# Adaptive Incident Correlation Engine (AICE)

AICE is a standalone cybersecurity component designed for **IoMT (Internet of Medical Things)** environments. It acts as an intelligent decision-making layer between AI vulnerability detection systems and SOC dashboards.

**Project Status**: Prototype / Demo Ready

## Features

- **Temporal Correlation**: Groups repeated alerts from the same device within configurable time windows (default 5 mins).
- **Adaptive Severity Scoring**: Calculates severity based on:
  - **Confidence** of the underlying ML prediction.
  - **Criticality** of the affected medical device (e.g., Ventilator > Printer).
  - **Frequency** of alerts (DDoS/Brute-force detection).
- **Compliance Tagging**: Automatically tags incidents with relevant regulations (HIPAA, SL-DPA).
- **False Positive Reduction**: Filters out isolated low-confidence noise.

## How to Run

1. **Generate Sample Data**:
   ```bash
   python tools/generate_data.py
   ```
   This creates `data/alerts.json` with 4 test scenarios (Critical Attack, Scan, Noise, etc.).

2. **Run the Engine**:
   ```bash
   python run_aice.py
   ```
   This processes the alerts and generates `data/incidents.json`.

3. **Check Output**:
   The script will print a summary. Full details are in `data/incidents.json`.

## Customization

- Edit `src/config.py` to change:
  - `TIME_WINDOW_MINUTES`: Duration for grouping alerts.
  - `DEVICE_CRIT_SCORES`: Criticality of devices.
  - `WEIGHT_*`: How severity is calculated.

## Integration

AICE accepts JSON input containing:
- `device_id`
- `timestamp`
- `prediction`
- `confidence_score`
- `device_type` (optional)

It outputs strictly formatted JSON `Incident` objects ready for SIEM/SOC ingestion.
